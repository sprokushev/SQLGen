Скрипты для взаимодействия с GIT

Актуальные версии - в https://git.promedweb.ru/rtmisdb/sqlgen/-/blob/current/Version

У них у всех примерно одинаковый принцип:
- Передаются схожие параметры, список и нзначение которых можно посмотреть в начале каждого cmd-файла или в настоящем файле или если запустить cmd-файл без параметров
- Результат выполнения возвращается через переменную окружения ERROR, значения также описаны в начале cmd-файла или в настоящем файле или если запустить cmd-файл без параметров
- В переменной BRANCH обычно возвращается текущая ветка, можно ее также проверять


git_newbranch.cmd
    Создаем новую ветку
    Если ветка уже есть - обновляем ее с удаленного репозитория и делаем в нее merge из ветки origin/master

    Параметры:
	%1 папка проекта GIT
	%2 ветка задачи
	%3 ветка от которой нужно создать новую ветку (не обязательный, по умолчанию master)
	%3, %4 или %5 NOUPPERCASE (не обязательный) - НЕ переводить принудительно в верхний регистр имя ветки задачи
	%3, %4 или %5 DEBUG (не обязательный) - режим отладки (пауза после каждой команды git)

    в %ERROR% возвращается результат: 
	= 0 (ok)
	= 10001 (неверные параметры запуска)
	= 10003 (ошибка команды GIT)
	= 10004 (ветка GIT не существует)
	= 99999 (неизвестная ошибка)

    в %BRANCH% возвращается текущая ветка

    Пример вызова:
	git_newbranch.cmd D:\Projects\dev_promed_pg PROMEDWEB-00000


git_add.cmd
    Добавить файлы в ветку задачи
    Ветка задачи должны быть текущей. Для смены ветки используем git_newbranch.cmd или git_switch.cmd

    Параметры:
	%1 папка проекта GIT
	%2 ветка задачи
	%3, %4, %5 NOUPPERCASE (не обязательный) - НЕ переводить принудительно в верхний регистр имя ветки задачи
	%3, %4, %5 DEBUG (не обязательный) - режим отладки (пауза после каждой команды git)
	%3, %4, %5 NOMERGEREQUEST (не обязательный) - без Merge Request

    в %ERROR% возвращается результат: 
	= 0 (ok)
	= 10001 (неверные параметры запуска)
	= 10002 (текущая ветка не соответствует ветке задачи)
	= 10003 (ошибка команды GIT)
	= 10004 (ветка GIT не существует)
	= 99999 (неизвестная ошибка)

    в %BRANCH% возвращается текущая ветка

    Пример вызова:
	git_add.cmd D:\Projects\dev_promed_pg PROMEDWEB-00000

git_merge.cmd
    Merge другой ветки в текущую
    Для смены текущей ветки используем git_switch.cmd 

    Параметры:
	%1 папка проекта GIT
	%2 ветка задачи, которую надо merge в текущую ветку
	%3, %4, %5 NOUPPERCASE (не обязательный) - НЕ переводить принудительно в верхний регистр имя ветки задачи
	%3, %4, %5 ORIGIN (не обязательный) - вливать из origin

    в %ERROR% возвращается результат: 
	= 0 (ok)
	= 10001 (неверные параметры запуска)
	= 10003 (ошибка команды GIT)
	= 10005 (ошибка при разрешении конфликта MERGE)
	= 10006 (требуется COMMIT)
	= 10007 (НЕ требуется COMMIT)
	= 10008 (ветка содержит BRANCH_DEV)
	= 99999 (неизвестная ошибка)

    в %BRANCH% возвращается текущая ветка

    Пример вызова:
	git_merge.cmd D:\Projects\dev_promed_pg PROMEDWEB-00000


Дополнительные скрипты:
git_touppercase.cmd - перевести строку в верхний регистр
git_run_merge.cmd - (для git_merge.cmd) выполнить команду из переменной %GITCMD% и проанализировать ее вывод на предмет ключевых слов-ошибок (error, fatal)
git_run.cmd - (для остальных cmd-файлов) выполнить команду из переменной %GITCMD% и проанализировать ее вывод на предмет ключевых слов-ошибок (error, fatal)
git_runsh.cmd - выполнить bash скрипт
git_exec.cmd - переключиться на папку с командным файлом, выполнить и вернуться в прежний каталог
git_find.cmd - вернуть список файлов, в имени которых встречается строка
git_lastcommit.cmd + git_lastcommit.sh - вернуть дату последнего комита
git_lastverintask.cmd + git-merged-ver.sh - вернуть последнюю версию влитую в текущую ветку
git_listbranch.cmd - вернуть список веток (всех)
git_listversion.cmd - вернуть список веток-версий
git_nocommit.cmd + git_nocommit.sh - проверить, есть ли отличия от индекса, требуется ли commit?
git_needpush.sh - список веток, для которых надо сделать push
git_nomergedversion.cmd + git-nomerged-ver.sh - вернуть список веток-версий еще не влитых в master
